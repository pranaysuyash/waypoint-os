# Jobs to be Done: All Personas

**Date**: 2026-04-14
**Purpose**: What is each person trying to accomplish? What problems do they need solved?

---

## Framework: Jobs to be Done (JTBD)

> "People don't buy products; they hire them to do a job."

Three dimensions:
1. **Functional job**: What they're trying to accomplish
2. **Emotional job**: How they want to feel
3. **Social job**: How they want to be perceived

---

## Persona 1: Agency Owner

### Profile

| Attribute | Description |
|-----------|-------------|
| **Role** | Owner/founder of travel agency (2-50 employees) |
| **Age** | 28-55 |
| **Mindset** | Growth-oriented, worried about scalability, time-starved |
| **Technical comfort** | Medium — uses business tools, not a developer |
| **Pain** | Can't scale, senior staff bottleneck, quality inconsistency |

---

### Jobs to be Done

#### Functional Jobs

| Job | When | Frequency | Pain if unsatisfied |
|-----|------|-----------|---------------------|
| **Scale planning capacity** | Want more revenue without proportional hiring | Daily | Revenue ceiling, burnout |
| **Standardize quality** | Junior staff produce inconsistent output | Weekly | Brand damage, refunds |
| **Train junior staff** | New agents need to get productive fast | Monthly | Slow ramp-up, mistakes |
| **Monitor team performance** | Need visibility into what's working | Weekly | Blind spots, can't improve |
| **Handle escalations** | Complex trips or upset clients | Weekly | Time drain, stress |
| **Make data-driven decisions** | Where to focus, what's working | Monthly | Guesswork, inefficiency |

#### Emotional Jobs

| Job | Description |
|-----|-------------|
| **Feel in control** | "I know what's happening in my business" |
| **Reduce anxiety** | "I'm not worried about quality dropping" |
| **Feel confident** | "I can take on more business without breaking" |
| **Reduce overwhelm** | "I'm not constantly putting out fires" |

#### Social Jobs

| Job | Description |
|-----|-------------|
| **Be seen as professional** | "We run a tight ship" |
| **Be seen as innovative** | "We use modern tools" |
| **Build trust with team** | "I invest in tools that make their lives easier" |

---

### Jobs Statement (Agency Owner)

> "Help me scale my agency without sacrificing quality or hiring more senior staff. I need to onboard juniors faster, ensure consistent output, and have visibility into operations — all while reducing my own firefighting."

| **Document everything** | For booking and records | Daily | Tedious, often forgotten |
| **Cross-sell/upsell** | Spot opportunities | Occasional | Missed revenue |

| Job | Description |
|-----|-------------|
| **Feel competent** | "I'm providing great service, not fumbling" |
| **Feel efficient** | "I'm handling my workload well" |
| **Feel proud** | "My clients get thoughtful, personalized options" |
| **Reduce stress** | "I'm not drowning in messages" |

#### Social Jobs

| Job | Description |
|-----|-------------|
| **Be seen as expert** | "I know my stuff, clients trust me" |
| **Be responsive** | "I reply quickly, clients appreciate that" |
| **Be thorough** | "I don't miss details" |

### Jobs Statement (Senior Agent)

> "Help me turn messy client messages into organized options without spending hours on back-and-forth. I need to extract intent, check feasibility, and create polished options — so I can focus on booking and building relationships instead of repetitive research."

## Cross-Persona Job Conflicts

### Where Jobs Conflict

| Conflict | Involved | Resolution |
|----------|----------|------------|
| **Speed vs. Quality** | Agent wants fast, Traveler wants thorough | AI does thorough work fast, Agent reviews |
| **Personalization vs. Scale** | Agency wants scale, Traveler wants personal | AI personalizes at scale using data |
| **Guidance vs. Autonomy** | Junior needs guidance, Senior wants autonomy | Tool provides both: suggested paths, optional detail |
| **Control vs. Convenience** | Owner wants control, Agent wants easy | Owner can configure, Agent gets defaults |

## Jobs Hierarchy (What Matters Most)

### For Agency Owner (Your Customer)

```
1. Scale capacity (growth constraint)
2. Standardize quality (brand protection)
3. Train juniors (operational efficiency)
4. Monitor performance (visibility)
5. Handle escalations (risk management)
```

### For Senior Agent (Primary User)

```
1. Extract intent efficiently (time savings)
2. Create good options (quality)
3. Check feasibility (avoid waste)
4. Follow up effectively (service)
5. Close bookings (revenue)
```

### For Junior Agent (Growth User)

```
1. Learn what to ask (competence)
2. Avoid mistakes (confidence)
3. Get unstuck (autonomy)
4. Create acceptable output (trust)
```

### For Traveler (End Beneficiary)

```
1. Feel understood (emotional connection)
2. See personalized options (value perception)
3. Make confident decision (reduce anxiety)
4. Get excited about trip (emotional payoff)
```

## Summary: Core Jobs Your Product Must Do

### Non-Negotiable (Must Do)

1. **Extract intent from messy messages** — Save agents time
2. **Flag contradictions and blockers** — Prevent wasted effort
3. **Generate quality options** — Maintain agency standards
4. **Provide rationale** — Help agents explain to travelers
5. **Enable junior independence** — Scale workforce
6. **Maintain agency control** — Keep human in the loop

### Important (Should Do)

7. **Track traveler preferences** — Personalization at scale
8. **Learn from agency patterns** — Improve over time
9. **Provide visibility** — Agency owner oversight
10. **Enable escalation** — Handle complex cases

### Nice to Have (Could Do)

11. **Booking integration** — Remove final step
12. **Multi-language support** — Expand market
13. **Mobile app** — Convenience for agents
14. **Analytics dashboards** — Business insights

## The "Jobs" Test for Every Feature

Before building anything, ask:

| Question | Yes → Build | No → Don't build |
|----------|------------|-----------------|
| Does it help a primary persona do a core job? | ✅ | ❌ |
| Does it solve a real pain (high frequency, high impact)? | ✅ | ❌ |
| Is it measurable (we'll know if it works)? | ✅ | ❌ |
| Can we execute it with current resources? | ✅ | Maybe defer |
| Does it align with our positioning? | ✅ | ❌ |

**Every feature should serve at least one job on the "Non-Negotiable" list.**
