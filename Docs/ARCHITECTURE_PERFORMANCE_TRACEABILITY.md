# Architectural Principle: Performance-to-Packet Traceability

**Date**: Wednesday, April 22, 2026
**Principle**: Data-Heavy Transparency & Drill-Down

In a high-stakes, decision-based agentic system, analytics must act as a gateway to operational transparency. Every metric shown on a dashboard is a lead, not just a fact. The interface must provide a seamless path from macro-performance trends down to the raw evidence supporting individual agent decisions.

---

## 1. The Traceability Hierarchy

We will implement a 3-tier navigation flow for all analytics and performance data:

1.  **Macro Tier (Insights Overview)**
    *   **Goal**: Monitor agency-wide throughput, quality, and autonomy health.
    *   **UI**: Charts (e.g., `TeamPerformanceChart`, `RevenueChart`).
    *   **Interaction**: Click a metric (e.g., "CSAT drop") or an agent row to enter the Meso Tier.

2.  **Meso Tier (Agent/Entity Performance)**
    *   **Goal**: Isolate specific operational patterns.
    *   **UI**: Agent-specific drill-down pages (`/owner/insights/[agentId]`).
    *   **Interaction**: View agent-specific decision logs, successful vs. overridden decisions, and time-series performance data. Click a specific decision or "Inquiry" to enter the Micro Tier.

3.  **Micro Tier (Packet & Provenance)**
    *   **Goal**: Verify the "Why."
    *   **UI**: Workspace View (Packet Inspector + Provenance Sidebar).
    *   **Interaction**: View the raw JSON data, evidence nodes, and decision reasoning generated by the agent. This is the source of truth.

---

## 2. Technical Roadmap for Traceability

### Phase A: Linking Logic
- [ ] Add `agentId` (or equivalent identifier) to `TeamMember` interface.
- [ ] Implement `next/link` wrappers around performance cards in `TeamPerformanceChart.tsx`.
- [ ] Establish URL pattern: `/owner/insights/[agentId]`.

### Phase B: Decision Contextualization
- [ ] Expose an endpoint `GET /api/agent/[agentId]/decisions` to fetch a paginated log of agent decisions.
- [ ] Create decision log view that displays:
    - Trip ID (Linked to workspace)
    - Decision State (e.g., `PROCEED_TRAVELER_SAFE`)
    - Confidence Score
    - Link to "View Evidence" (opens the workspace in read-only audit mode).

### Phase C: Evidence Traceability
- [ ] Build "Audit Overlay" or "Read-only Workspace View" that renders the `PacketPanel` for any historical trip ID.
- [ ] Enable the Provenance Sidebar for historical packets, showing exactly what evidence supported that decision.

---

## 3. Why This Wins
- **Quality Assurance**: Agencies can audit decisions during human-in-the-loop interventions.
- **Continuous Learning**: By identifying the *specific* trip where an agent performed poorly, we can capture that as a test fixture for future evaluations.
- **Agency Trust**: Transparency builds confidence. Showing the "work" behind the decision turns the agent from a "black box" into a "transparent partner."
