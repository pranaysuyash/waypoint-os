# User Journeys and Aha Moments

**Date**: 2026-04-14
**Purpose**: Map each persona's journey from frustration to delight

---

## Journey Framework

Each journey covers:
1. **Before** — Current state, pain points
2. **Trigger** — What starts the journey
3. **During** — Steps, emotions, friction
4. **Aha Moment** — When they realize value
5. **After** — New state, benefits

## What Makes the Aha Moment Possible?

### 1. Speed (Agent Side)

> "I expected this to take hours. It took minutes."

**How**: AI does the heavy lifting, human reviews and tweaks.

---

### 2. Quality (Agent Side)

> "The output is better than I would have done myself."

**How**: AI is thorough, doesn't forget things, explains reasoning.

---

### 3. Learning (Junior Side)

> "I understand WHY these questions matter."

**How**: Explanations, rationale, "why this matters" notes.

---

### 4. Personalization (Traveler Side)

> "They actually heard what we said."

**How**: Intent capture that preserves nuance, not just categories.

---

### 5. Control (Agency Owner Side)

> "I can see what's happening, and it's good."

**How**: Dashboards, logs, transparency.

---

## Design Implications

### For Senior Agents

| Design goal | How |
|-------------|-----|
| **Make speed visible** | Show processing time: "Processed in 45 seconds" |
| **Enable quick review** | Scannable output, key flags highlighted |
| **Allow easy tweaks** | One-click edits, not rebuilds |

### For Agency Owners

| Design goal | How |
|-------------|-----|
| **Make quality visible** | Dashboards, before/after metrics |
| **Show learning curve** | Junior progress over time |
| **Enable oversight** | View any trip, see agent decisions |

### For Junior Agents

| Design goal | How |
|-------------|-----|
| **Teach, don't just tell** | Explanations for every suggestion |
| **Show what good looks like** | Example outputs, rationale |
| **Build confidence** | Gradual autonomy, guardrails stay available |

### For Travelers

| Design goal | How |
|-------------|-----|
| **Show you heard them** | Echo back key preferences in options |
| **Explain the trade-offs** | Why this option vs. that one |
| **Make decisions easy** | Clear comparisons, not overwhelming data |
