# Feature: AI Agent Team Orchestrator

## POV: System Core / Operator POV

### 1. Objective
To coordinate a "Workforce" of specialized AI agents (Shadow Agents) that handle discrete tasks (Sourcing, Compliance, Support, Intel) in parallel, presenting a unified outcome to the human agent.

### 2. Functional Requirements

#### A. The Specialist Registry (Shadow Agents)
- **The "Sourcing Specialist"**: Dedicated to finding the deepest net rates across fragmented BedBanks and DMCs.
- **The "Compliance Guardian"**: Focused solely on ADM prevention, GDS rules, and Visa legality.
- **The "Vibe Auditor"**: Focused on the "Taste Graph" and "Visual Intent" to ensure the aesthetics are correct.
- **The "Logistics Pilot"**: Focused on transfers, wait times, and "Last Mile" connectivity.

#### B. Collaborative "Blackboard" Architecture
- **Shared Working Memory**: All specialist agents contribute to a "Draft PNR" on a shared blackboard.
- **Conflict Resolution**: If the "Sourcing Specialist" finds a cheap hotel but the "Compliance Guardian" flags it as "High Risk," the Orchestrator forces a "Debate" or escalates to a Human Senior Agent.

#### C. Intelligent Task Delegation
- **Asynchronous Execution**: While the traveler is chatting with the "Concierge Agent," the "Sourcing Agent" is already pre-fetching hotel options in the background.
- **Context Hand-off**: Ensuring that if a task moves from the "Support Agent" to a "Human Specialist," the full chain of AI-reasoning is summarized and visible.

### 3. Core Engine Logic
- **"The Master Orchestrator"**: An LLM-based controller that decides which specialist agent to trigger based on the "Customer Intent."
- **Feedback Loops**: Each specialist agent learns from Human "Corrections" (e.g., if a Human rejects a sourcing option, the Sourcing Agent updates its internal weights for that customer).

### 4. Safety & Scalability
- **Token Budgeting**: Monitoring the "Cost-per-Booking" for AI compute to ensure the "Shadow Workforce" doesn't erode the agency's margin.
- **Agent Sandbox**: Testing new "Specialist Agents" (e.g., a "Ski Specialist") in a shadow mode before they affect real PNRs.
