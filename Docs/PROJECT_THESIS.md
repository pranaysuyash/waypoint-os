# Project Thesis: Boutique Travel Agency AI Ops

## Core Philosophy
The project is an AI-assisted operating system for boutique travel agencies. It is a workflow compression tool for solo/small agencies, not a generic consumer trip planner.

### Key Optimization Objectives
- **Client Fit**: Moving from group-level planning to per-person suitability (e.g., toddler/elderly suitability for specific activities).
- **Conversion Likelihood**: Reducing friction from lead to quote.
- **Margin Quality**: Ensuring the package is commercially viable for the agency.
- **Operational Ease**: Prioritizing a sourcing hierarchy: Preferred Network $\rightarrow$ Network/Consortium $\rightarrow$ Open Market.
- **Risk Reduction**: Detecting "wasted spend" and pacing errors before the client sees the itinerary.

## The Agency Wedge
The product targets the "one-man show" or small boutique agency that currently relies on WhatsApp, Google Sheets, and mental memory. The goal is to provide a "Copilot" that handles the repetitive research and coordination, leaving the human to manage the relationship.

## The Hybrid Validation Model
A free "Intelligence Layer" (Itinerary Audit) is used as a lead-gen tool. It allows users to upload existing plans (PDFs, URLs) to identify risks, wasted spend, and fit issues, which then converts them into high-quality leads for agency partners.
