MULTI-AGENT INFRASTRUCTURE — WAYPOINT OS
=========================================

This document was originally written before ADR-006 was accepted.  It captures
the long-term product vision.  See `Docs/architecture/MULTI_AGENT_RUNTIME_ROADMAP.md`
for the current roadmap and `Docs/status/MULTI_AGENT_RUNTIME_IMPLEMENTATION_TRACKER_2026-05-14.md`
for what has been built.

─────────────
PRODUCT THESIS (still valid)
─────────────

Waypoint OS should become a multi-agent travel operating system where
specialized product agents (Scout, Communicator, QA, Committee, Operator
Copilot) use the deterministic spine as a pure judgment tool.  End users
interact with these agents through the workbench.

─────────────
CURRENT IMPLEMENTATION (ADR-006)
─────────────

The repo now has (instead of orchestrator.py + base_agent.py):

- 16 static in-repo agents registered via build_default_registry()
- AgentSupervisor with scan/execute passes and audit events
- InMemoryWorkCoordinator and SQLWorkCoordinator (both tested)
- SpineRequeuePort with Disabled and Inline implementations
- RecoveryAgent using the port abstraction
- Config factory with env-var validation
- Runtime endpoint exposing health and config
- 117 agent-related tests

See `Docs/status/MULTI_AGENT_RUNTIME_IMPLEMENTATION_TRACKER_2026-05-14.md`.

─────────────
VALID ROADMAP AGENTS (not yet implemented)
─────────────

These remain valid product direction:

- Communicator Agent: blocked state → clarification drafts
- Scout Agent: proactive info retrieval (visa, weather, safety)
- QA Agent: analyze failures, propose fixtures and tests
- Committee System: Budget Optimizer + Experience Maximizer + Trip Architect
- Operator Copilot: natural-language intervention on canonical packet

─────────────
SUPERSEDED IMPLEMENTATION ASSUMPTIONS
─────────────

These were described as "Phase 1 Foundation" but are no longer the approach:

- src/agents/orchestrator.py as mandatory first file
- src/agents/base_agent.py as mandatory first file
- Dynamic discovery as near-term registry mechanism
- Message bus before durable work coordination exists
- Agent memory before clear state model exists
- Resource isolation before job execution model exists
