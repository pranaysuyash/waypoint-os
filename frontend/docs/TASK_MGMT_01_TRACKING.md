# Task Management & Assignment — Task Tracking

> Research document for trip-related task management, checklists, and workload tracking.

---

## Key Questions

1. **How do we break trip work into trackable tasks with clear ownership?**
2. **What task types exist across the trip lifecycle?**
3. **How do we handle recurring tasks (visa checks, payment reminders)?**
4. **What's the task dependency model?**
5. **How do we visualize task progress across trips and agents?**

---

## Research Areas

### Task Model

```typescript
interface TripTask {
  taskId: string;
  tripId: string;
  parentTaskId?: string;             // For subtasks
  title: string;
  description: string;
  type: TaskType;
  status: TaskStatus;
  priority: TaskPriority;
  assignee: string;
  createdBy: string;
  createdAt: Date;
  dueDate: Date;
  completedAt?: Date;
  dependencies: TaskDependency[];
  checklist: ChecklistItem[];
  tags: string[];
  estimatedMinutes: number;
  actualMinutes?: number;
  autoGenerated: boolean;           // System-created vs. manual
}

type TaskType =
  | 'customer_communication'         // Call customer, send quote
  | 'supplier_interaction'           // Book hotel, confirm flight
  | 'document_preparation'           // Generate itinerary, visa docs
  | 'payment_processing'             // Collect payment, issue invoice
  | 'internal_review'                // Quality check, compliance review
  | 'data_entry'                     // Enter traveler details, passport info
  | 'coordination'                   // Arrange transfer, coordinate guide
  | 'follow_up'                      // Confirm booking, check status
  | 'escalation'                     // Owner review, risk acceptance
  | 'post_trip'                      // Feedback collection, document archive
  | 'administrative';                // Internal admin, reporting

type TaskStatus =
  | 'pending'
  | 'in_progress'
  | 'blocked'
  | 'waiting_external'              // Waiting on customer or supplier
  | 'completed'
  | 'cancelled'
  | 'overdue';

type TaskPriority =
  | 'critical'              // Trip is blocked until this is done
  | 'high'                  // Must complete today
  | 'medium'                // Should complete within 2 days
  | 'low';                  // Nice to have, no deadline pressure
```

### Task Dependencies

```typescript
interface TaskDependency {
  dependsOnTaskId: string;
  dependencyType: DependencyType;
}

type DependencyType =
  | 'finish_to_start'        // Task B can't start until Task A finishes
  | 'start_to_start'         // Task B can't start until Task A starts
  | 'finish_to_finish'       // Task B can't finish until Task A finishes
  | 'start_to_finish';       // Task B can't finish until Task A starts

// Dependency examples for a trip:
// "Collect payment" (finish_to_start) → "Book hotel"
// "Get passport copies" (finish_to_start) → "Apply for visa"
// "Book flights" (start_to_start) → "Arrange airport transfer"
// "Confirm hotel" (finish_to_start) → "Send itinerary to customer"

// Auto-generated task chains:
// New trip created → Generate standard task chain based on trip type:
//   1. Customer acknowledgment (auto-assigned to intake agent)
//   2. Requirements extraction (auto-assigned to intake agent)
//   3. Flight search (auto-assigned based on expertise)
//   4. Hotel search (auto-assigned based on expertise)
//   5. Itinerary assembly (depends on 3+4)
//   6. Pricing calculation (depends on 5)
//   7. Quote delivery (depends on 6)
//   8. Customer follow-up (3 days after 7)
//   9. Payment collection (after customer approval)
//   10. Booking confirmation (depends on 9)
//   11. Document preparation (depends on 10)
//   12. Pre-departure check (7 days before travel)
```

### Task Views

interface TaskView {
  viewType: TaskViewType;

type TaskViewType =
  | 'my_tasks'               // Tasks assigned to current agent

## Open Problems

1. **Task granularity** — Too fine-grained = overhead. Too coarse = things fall through cracks. Need right-sized task decomposition per trip type.

2. **Auto-generation vs. manual** — System-generated tasks are great for standard flows but can't handle edge cases. Need auto-generation with manual override.

3. **Overdue cascading** — One overdue task blocks dependent tasks. Need escalation when a blocking task is overdue (reassign or split).

4. **Task completion quality** — Marking a task "done" doesn't mean it was done well. Need quality checks for critical tasks (visa docs, payment amounts).

5. **Cross-trip tasks** — Some tasks span multiple trips (bulk hotel negotiation, seasonal rate updates). Need a way to manage non-trip-specific work.
