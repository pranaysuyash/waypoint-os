```
3. Select and Book
```

```typescript
// Research-level model - not final

interface ShoreExcursion {
  id: string;
  portId: string;
  name: string;

  // Details
  description: string;

  physicalLevel: PhysicalLevel;

  // Logistics
  startTimes: string[];

  provider: ExcursionProvider;

  | 'sightseeing'

  | 'transportation';

  | 'active'
  | 'strenuous';

  | 'refunded';

interface ExcursionProvider {
  id: string;
  name: string;
  type: 'cruise_line' | 'independent' | 'platform';
  contactInfo?: string;
```

## Open Problems

### 1. Price Disparity
**Challenge:** Independent excursions are much cheaper

**Options:**
- Show both options transparently
- Explain cruise line advantages
- Let customer decide
- Offer best price guarantee?

### 2. Capacity Management
**Challenge:** Popular excursions sell out quickly

**Options:**
- Pre-booking with priority
- Waitlist management
- Multiple time slots
- Similar alternatives

### 3. No-Show Handling
**Challenge:** Customers book but don't show up

**Options:**
- Strict no refund policy
- Reminder notifications
- Easy cancellation window
- Credit for future use

### 4. Quality Assurance
**Challenge:** Can't verify independent operator quality

**Options:**
- Reviews and ratings
- Vetted partners only
- Dispute resolution
- Insurance backup
