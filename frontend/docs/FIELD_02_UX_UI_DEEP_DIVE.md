## 1. Design Philosophy

### 1.1 Core UX Principles

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         FIELD EDITING UX PRINCIPLES                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  🔹 CLARITY                                                                 │
│      └─ Field purpose is immediately obvious                                │
│      └─ Labels are clear and concise                                        │
│      └─ Help text is available when needed                                  │
│                                                                             │
│  🔹 FEEDBACK                                                                │
│      └─ Validation errors are shown immediately                             │
│      └─ Success states are communicated                                     │
│      └─ Loading states are visible                                          │
│      └─ Dirty state is clearly indicated                                    │
│                                                                             │
│  🔹 EFFICIENCY                                                              │
│      └─ Autocomplete reduces typing                                         │
│      └─ Smart defaults speed entry                                          │
│      └─ Keyboard navigation is supported                                    │
│      └─ Bulk editing is available where appropriate                         │
│                                                                             │
│  🔹 FORGIVENESS                                                             │
│      └─ Undo/redo is available                                              │
│      └─ Confirmation for destructive actions                                │
│      └─ Auto-save prevents data loss                                        │
│      └─ Recovery from errors is easy                                       │
│                                                                             │
│  🔹 CONSISTENCY                                                             │
│      └─ All fields follow the same patterns                                 │
│      └─ Validation messages are consistent                                  │
│      └─ Interaction patterns are predictable                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 Mental Model

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            USER MENTAL MODEL                                │
└─────────────────────────────────────────────────────────────────────────────┘

    "I need to update the customer's email address"

            │
            ▼
    ┌───────────────┐
    │  FIND FIELD   │ ──► Scanning for "Email" label
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │  CLICK FIELD  │ ──► Focus, see current value
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │  EDIT VALUE   │ ──► Type new email
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │  SEE FEEDBACK │ ──► Validation passes/fails
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │  MOVE ON      │ ──► Tab to next field or click away
    └───────────────┘
```

### 7.2 ARIA Attributes Reference

```typescript
/**
 * ARIA attributes for field components
 */
interface FieldAriaAttributes {
  // Labeling
  'aria-label'?: string;
  'aria-labelledby'?: string;
  'aria-describedby'?: string;

  // State
  'aria-invalid'?: boolean | 'false' | 'true' | 'grammar' | 'spelling';
  'aria-required'?: boolean;
  'aria-disabled'?: boolean;
  'aria-readonly'?: boolean;

  // Values
  'aria-valuenow'?: number;
  'aria-valuetext'?: string;
  'aria-valuemin'?: number;
  'aria-valuemax'?: number;

  // Live regions (for errors, validation)
  'aria-live'?: 'off' | 'polite' | 'assertive';
  'aria-atomic'?: boolean;
  'aria-busy'?: boolean;

  // Combobox specific
  'aria-expanded'?: boolean;
  'aria-haspopup'?: boolean | 'listbox' | 'menu' | 'dialog';
  'aria-autocomplete'?: 'none' | 'inline' | 'list' | 'both';
  'aria-controls'?: string;
  'aria-activedescendant'?: string;

  // Options
  'aria-selected'?: boolean;
  'aria-checked'?: boolean | 'mixed';
}

/**
 * ARIA roles
 */
enum AriaRole {
  // Form roles
  FORM = 'form',
  GROUP = 'group',
  SEARCH = 'search',

  // Input roles
  TEXTBOX = 'textbox',
  COMBOBOX = 'combobox',
  LISTBOX = 'listbox',
  OPTION = 'option',
  SPINBUTTON = 'spinbutton',
  SLIDER = 'slider',

  // Status roles
  ALERT = 'alert',
  STATUS = 'status',
  PROGRESSBAR = 'progressbar',

  // Button roles
  BUTTON = 'button'
}
```
