Architecture

These models sit above the CanonicalPacket as the institutional memory layer. They turn one-off trips into repeatable DNA.

KNOWLEDGE LAYER → Operational Intelligence

The CanonicalPacket captures what is happening now. The Knowledge Layer captures what we've learned from 1,000+ past trips. Two core models power this: TemplateGenome (the DNA of successful trips) and SupplierProfile (who actually delivers).

NB02 Template Suggestion — live example 
# NB02: Template Suggestion Engine
def suggest_for_packet(packet: CanonicalPacket) -> TemplateSuggestion:
    """
    When packet has destination=Singapore, pax=4, profile=family,
    NB02 queries templates, finds singapore_family_v3 with 0.92 match,
    suggests it, predicts 4.6 satisfaction and 19% margin
    """
    candidates = template_store.query(
        destination=packet.destination,  # "SIN"
        pax_count=packet.pax_count,      # 4
        profile=packet.client_profile   # "family"
    )
    
    best = max(candidates, key=lambda t: t.match_score)  # 0.92
    
    # Enrich with supplier reliability
    supplier_boost = supplier_graph.get_reliability(
        best.supplier_preferences["hotel"][0]
    )  # Marina Bay: 0.94
    
    return TemplateSuggestion(
        template_id="singapore_family_v3",
        match_score=0.92,
        confidence="high",
        predicted_satisfaction=4.6,
        predicted_margin_pct=19.0,
        predicted_issues=["hotel_checkin_delay: 23%"],
        reasoning=f"47 prior uses, {supplier_boost:.2f} supplier reliability, 94% success rate",
        required_customizations=["kids_aged_8_11 → add Universal Studios"]
    )
